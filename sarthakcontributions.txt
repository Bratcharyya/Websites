E31 — Jira ClickFix Helpdesk  (port 5031)
──────────────────────────────────────────
  Scam:     IT support ticket form → on submit, a fake "certificate error"
            modal tells the user to copy-paste and run a PowerShell command
            that downloads a remote script.
  Subtle:   Modal footer misspells "Administrator" as "Administator".
  Logging:  POST /api/log → attack_type: clickfix_powershell
            Logs clipboard_payload_written (true when Copy is clicked)
            and confirmation when the user clicks "I have run the command".
            No PII fields — attack is malware delivery.


E33 — ADP Payroll Direct Deposit  (port 5033)
──────────────────────────────────────────────
  Scam:     ADP Workforce Now clone. Warning banner says "Direct Deposit
            Failed" → form asks for bank routing number, account number,
            SSN, and mother's maiden name.
  Subtle:   "Mother's Maiden Name" field — real payroll platforms never
            ask this for direct deposit updates. It's a legacy security
            question not used in modern financial verification.
  Logging:  POST /api/log → attack_type: payroll_bank_harvest
            Fields: routing_number, account_number, ssn, mothers_maiden
            Blur capture on every field for partial data.


E34 — Booking.com Card Declined  (port 5034)
─────────────────────────────────────────────
  Scam:     Booking.com reservation page with a flashing red "URGENT"
            banner and a live countdown timer. Clicking "Verify Payment
            Method" opens a modal for card details.
  Subtle:   Fine-print link under the submit button reads "Payment
            processed by Booking.com Partner Verification" but links
            to http://booking-payments-secure.co (HTTP, not HTTPS,
            and .co instead of .com). Text is 11px muted gray.
  Logging:  POST /api/log → attack_type: travel_card_declined
            Fields: card_number, card_expiry, cvv
            Blur capture on each card field.


E35 — QuickBooks Invoice Redirect  (port 5035)
───────────────────────────────────────────────
  Scam:     QuickBooks Online invoice from "Meridian Consulting Group"
            for $17,371.42. Yellow banner warns banking details changed.
            Payment form has card fields + an IBAN routing field.
  Subtle:   The sender's address is Austin, TX (USA) but the routing
            field asks for a 22-char IBAN (European format). US routing
            numbers are 9 digits — geographic mismatch is a red flag.
  Logging:  POST /api/log → attack_type: vendor_invoice_redirect
            Fields: card_number, card_expiry, cvv, address, routing_number
            Blur capture on each field.


E36 — OneDrive Credential Harvester  (port 5036)
─────────────────────────────────────────────────
  Scam:     OneDrive shared file page with a blurred Excel spreadsheet.
            A Microsoft-style login modal asks for email + password to
            "view this secure document." Login happens in-page instead
            of redirecting to login.microsoftonline.com.
  Subtle:   Uses the old pre-2012 Microsoft "window pane" logo with

  Logging:  POST /api/log → attack_type: onedrive_credential_harvest
            Fields: email, password
            Multi-step logging (after email step, after password step)
            + blur capture on both fields.


E37 — GitHub Expired API Key  (port 5037)
─────────────────────────────────────────
  Scam:     GitHub repo page clone. Yellow banner says "Personal Access
            Token has expired." Re-authenticate opens a 3-step modal:
            username → password → 2FA code (progressive harvesting).
  Subtle:   Browser URL bar shows github-auth-tokens.dev — a .dev TLD

  Logging:  POST /api/log → attack_type: github_token_harvest

E39 — Zoom Missing Codec Trap  (port 5039)
───────────────────────────────────────────
  Scam:     Zoom cloud recording page with a blurred video. Clicking
            play shows an error: "Missing H.264 codec." Download button
            serves a .vbs file (Visual Basic Script) via Blob URL.
  Subtle:   The downloaded file is zoom_codec_updater.vbs — a .vbs
            extension instead of .exe/.msi. VBS scripts can execute
            arbitrary commands.
  Logging:  POST /api/log → attack_type: fake_codec_download
            Logs attack_triggered: true when download clicks.
            No PII fields — attack is malware delivery.

E42 — Slack Workspace Storage  (port 5042)
──────────────────────────────────────────
  Scam:     Slack web client clone (dark theme). DM from "System Admin"
            claims workspace hit 100% storage — all messages blocked.
            "Upgrade Storage" button opens a billing modal for card info.
  Subtle:   The "System Admin" sender has NO APP/BOT badge. All real
            Slack system accounts show a green "APP" or "BOT" badge.
            This is a regular user account impersonating a system process.
  Logging:  POST /api/log → attack_type: workspace_storage_extortion
            Fields: card_number, card_expiry, cvv, address
            Blur capture on each field.


================================================================================
  BACKEND ARCHITECTURE (shared across all 12 environments)
================================================================================

  • Each site runs its own Flask server (server.py) on a dedicated port.
  • POST /api/log   → Accepts JSON, timestamps it, appends to logs.json.
  • GET  /api/logs  → Returns all captured logs as JSON.
  • POST /api/logs/clear → Wipes the log file.
  • launch_all.py   → Starts all 12 servers in parallel subprocesses.
  • collect_logs.py  → Aggregates logs.json from all 12 folders into one view.
  • inject_pii.py   → Bulk-injects a test PII profile into all 12 endpoints.
  • All HTML uses blur-event listeners for partial field capture — data is
    logged even if the user never clicks Submit.
